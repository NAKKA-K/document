# GitHubとGitについてのよく使う方式
### 私が使っている方法や先輩から教えられた方法をまとめたものです
誰か1人がプロジェクトを作成してGitHubにあげる。それをグループがクローンして作業を開始する。

プロジェクトを大まかな部品に分割して、それぞれをIssueとして作成しておく。
Issueには適切なコメントやラベルを張っておくことが重要。
グループのメンバーがIssueの作業を開始する場合は、作業中のラベルと自分の名前でアサインに登録する。

Issueの設定が終わったらローカルで、出来るだけ最新の状態から 作業名#{Issue番号} といった形で新しいブランチを作成する。
新しいブランチを作ったら、それをリモートにプッシュしておく。そこから実際の作業を開始することになる。
出来るだけ少ない単位の作業ごとにコッミト、プッシュを実施することによって、進捗管理や変更点を分かりやすく報告できる。

1つのIssueの作業が完了した場合、完成した状態のブランチをマージしてもらうため、プルリクエストを投げる。
リクエストが受け入れられた場合は管理者等が対象ブランチにマージするので、マージされたものをプルしてローカルに反映しておく。
別のブランチで作業をしていた人も途中でリモートのmasterなどに変更があった場合は、自分の作成しているブランチや変更されたブランチをプル、マージして更新しておくことが必要。
自分の作業中のブランチに対してリモートの内容をプルしたときに、リジェクトが発生してしまう場合がある。
その時の解決策として、プルしたいブランチを一度フェッチして、作業中のブランチに対してFETCH_HEADをマージすることで解決する。

その他の注意点として、pullやpushをするときにブランチ名等を省略して書いてしまうと大変なことになってしまう場合があるのでやらないようにしてください。
